# I18n design — svenska/engelska

**Datum:** 2026-03-31
**Issue:** #30

## Mål

Stöd för svenska och engelska i Lagret-appen (web + React Native). Språket detekteras automatiskt från webbläsarens/enhetens inställningar med möjlighet att byta manuellt i hushållsinställningar. Valet sparas i localStorage.

## Tekniskt val

- **Bibliotek:** react-i18next (fungerar i React web och React Native)
- **Filer:** `src/i18n/sv.json` och `src/i18n/en.json` — en fil per språk, inga namespaces
- **Initiering:** `src/i18n/index.ts` — konfigurerar i18next med localStorage-persistence och browser language detection
- **Dayjs:** Byt locale dynamiskt när språket byts (`dayjs.locale('sv')` / `dayjs.locale('en')`)

## Migrationsstrategi

Inkrementell — en feature-area i taget. Varje steg resulterar i ett eget PR.

### Steg 1 — Infrastruktur
- Installera `react-i18next`, `i18next`, `i18next-browser-languagedetector`
- Skapa `src/i18n/index.ts` med konfiguration
- Skapa `sv.json` och `en.json` med `common`-sektionen (knappar, labels, fel som återkommer överallt)
- Importera i18n i `main.tsx`

### Steg 2 — Gemensamma strängar
Extrahera strängar som används i flera komponenter:
- Formulärlabels: Namn, Antal, Enhet, Förvaringsplats, Bäst-före datum, Kategori
- Knappar: Lägg till, Spara, Avbryt, Bekräfta, Logga in, Logga ut, Skapa, Gå med
- Fel från stores: "Inget hushåll laddat", "Okänt fel"
- Kategorier (`src/lib/categories.ts`)
- Enhetsgrupper (`src/lib/units.ts`)

### Steg 3–7 — Sida för sida
Prioriteringsordning baserat på antal strängar:
3. HouseholdPage (~65 strängar)
4. ShoppingListPage (~35 strängar) + AddItemModal + EditItemModal
5. RecipesPage (~38 strängar) + CookingMode
6. InventoryPage (~18 strängar) + ItemCard + Scanner
7. LoginPage (~24 strängar) + AppLayout nav + NotificationBanner + OfflineBanner

### Steg 8 — Språkväljare
- Lägg till language toggle i HouseholdPage (inställningssektionen)
- Spara val i localStorage (`lagret:language`)

## Streckkodsdatabasen

Lägg till `name_en` kolumn i `barcodes`-tabellen (migration). Befintliga poster behåller `name` (svenska) och `name_en` läggs till som nullable. Appen visar rätt namn baserat på aktivt språk, med fallback till det andra språket om översättning saknas.

```sql
ALTER TABLE barcodes ADD COLUMN name_en text;
```

## Pluralisering

i18next hanterar pluralisering via `_one`/`_other`-suffix:
```json
"itemCount": "{{count}} vara",
"itemCount_other": "{{count}} varor"
```

Relevanta ställen: "X vara(or)", "X dag(ar) kvar", "X ingrediens(er)".

## Vad ingår inte

- Översättning av receptnamn och ingredienser från TheMealDB (dessa är alltid engelska från API:t)
- Översättning av befintliga produktnamn i `barcodes`-tabellen (hanteras vid skanning framöver)
- RTL-stöd
