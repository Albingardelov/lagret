# I18n (Svenska/Engelska) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Swedish/English support to Lagret using react-i18next, detected from browser language with a manual override saved in localStorage.

**Architecture:** A single `src/i18n/index.ts` initialises i18next with browser-language detection and localStorage persistence. Two translation files (`sv.json`, `en.json`) hold all UI strings. Categories and units keep their Swedish DB values but are displayed via translation lookup. Each migration task ends with a commit.

**Tech Stack:** react-i18next, i18next, i18next-browser-languagedetector, dayjs locale switching

- [ ] **Step 4: Create i18n initialisation**

Create `src/i18n/index.ts`:

```ts
import i18n from 'i18next'
import { initReactI18next } from 'react-i18next'
import LanguageDetector from 'i18next-browser-languagedetector'
import dayjs from 'dayjs'
import 'dayjs/locale/sv'
import 'dayjs/locale/en'

import sv from './sv.json'
import en from './en.json'

i18n
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    resources: { sv: { translation: sv }, en: { translation: en } },
    fallbackLng: 'sv',
    supportedLngs: ['sv', 'en'],
    detection: {
      order: ['localStorage', 'navigator'],
      lookupLocalStorage: 'lagret:language',
      caches: ['localStorage'],
    },
    interpolation: { escapeValue: false },
  })

// Sync dayjs locale with i18n language
const syncDayjs = (lang: string) => dayjs.locale(lang.startsWith('en') ? 'en' : 'sv')
syncDayjs(i18n.language)
i18n.on('languageChanged', syncDayjs)

export default i18n
```

```ts
import { describe, it, expect } from 'vitest'
import sv from './sv.json'
import en from './en.json'

function getLeafKeys(obj: Record<string, unknown>, prefix = ''): string[] {
  return Object.entries(obj).flatMap(([key, val]) => {
    const full = prefix ? `${prefix}.${key}` : key
    if (val !== null && typeof val === 'object' && !Array.isArray(val)) {
      return getLeafKeys(val as Record<string, unknown>, full)
    }
    return [full]
  })
}

describe('i18n parity', () => {
  it('sv.json and en.json have identical keys', () => {
    const svKeys = getLeafKeys(sv as unknown as Record<string, unknown>).sort()
    const enKeys = getLeafKeys(en as unknown as Record<string, unknown>).sort()

    const missingInEn = svKeys.filter((k) => !enKeys.includes(k))
    const missingInSv = enKeys.filter((k) => !svKeys.includes(k))

    expect(missingInEn, 'Keys in sv.json but missing in en.json').toEqual([])
    expect(missingInSv, 'Keys in en.json but missing in sv.json').toEqual([])
  })
})
```

- [ ] **Step 3: Update barcodeRegistry.ts to handle name_en**

Replace the contents of `src/lib/barcodeRegistry.ts`:

```ts
import { supabase } from './supabase'

export interface BarcodeEntry {
  name: string
  nameEn?: string
  unit: string
  category?: string
}

export async function lookupBarcodeRegistry(barcode: string): Promise<BarcodeEntry | null> {
  const { data } = await supabase
    .from('barcodes')
    .select('name, name_en, unit, category')
    .eq('barcode', barcode)
    .maybeSingle()
  if (!data) return null
  return { name: data.name, nameEn: data.name_en ?? undefined, unit: data.unit, category: data.category ?? undefined }
}

export async function saveBarcodeRegistry(barcode: string, entry: BarcodeEntry): Promise<void> {
  const {
    data: { user },
  } = await supabase.auth.getUser()
  await supabase
    .from('barcodes')
    .upsert(
      { barcode, name: entry.name, name_en: entry.nameEn ?? null, unit: entry.unit, category: entry.category, created_by: user?.id },
      { onConflict: 'barcode', ignoreDuplicates: true }
    )
}
```
